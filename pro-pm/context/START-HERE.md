# Start Here (60 Seconds)

You can use Pro PM **right now** with zero setup.

## Option A: Zero setup (recommended first try)

1. Wire Pro PM into your AI tool → see `INSTALL.md` and `platform-adapters/universal-setup.md` (works on Cursor, Claude Code, Codex, Windsurf, or any agent)
2. Ask anything in plain language:

   - *"Write a PRD for bulk export for enterprise admins"*
   - *"Prep me for a roadmap review with my VP"*
   - *"Draft a Slack update on sprint progress"*
   - *"Help me think through pricing for a new AI feature"*

Pro PM uses built-in defaults from [`_defaults.md`](_defaults.md). Assumptions are labeled; you correct inline if needed.

## Option B: 2-minute personalization (optional)

Fill **only** these two fields — everything else can wait:

**[`context/me/profile.md`](me/profile.md)** — Role + domain (2 lines)

**[`context/company/overview.md`](company/overview.md)** — Company name + stage (2 lines)

That’s enough for noticeably better tailoring.

## Option C: Go deeper over time

Add when useful (never required upfront):

| When | File |
|------|------|
| You own a product | Copy `products/_template/` → `products/your-product/` |
| You email specific leaders | `people/_person-template/` |
| You use Jira/Slack/etc. | Matching file in `integrations/` |
| Big initiative | `initiatives/_template.md` |
| Planning / capacity | `team/team.md` or `team/capacity.md` (optional) |

## What Pro PM will NOT do

- Make you fill long forms before helping
- Ask 10 discovery questions on every task
- Refuse work because context is missing
- Require API keys to draft artifacts

## What makes output sharper later

Context files, stakeholder personas, and integration configs — add them incrementally after you see value.
