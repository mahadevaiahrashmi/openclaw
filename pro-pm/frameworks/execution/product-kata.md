# Product Kata

## What It Is
The Product Kata, adapted by Melissa Perri from Toyota's Improvement Kata, is a structured approach to product development that keeps teams focused on outcomes through a repeating cycle: understand the direction, grasp the current condition, set the next target condition, and experiment toward it. It prevents teams from jumping to solutions without understanding where they are and where they need to go.

## When to Use
- When teams are shipping features but not moving metrics
- As a daily/weekly operating rhythm for outcome-focused product work
- Coaching PMs to think in outcomes and experiments instead of feature lists
- When the team feels stuck and needs a structured way to make progress

## When NOT to Use
- Very early-stage exploration where you don't have enough data for "current condition"
- Crisis response where you need to act immediately
- Simple, well-understood work that doesn't need iterative experimentation

## How to Apply
1. **Understand the Direction**: What is the overall vision and strategic intent? What does success look like? (Informed by company strategy, OKRs, North Star Metric)
2. **Grasp the Current Condition**: Where are you now? What do the metrics say? What do customers tell you? What have you learned from recent experiments?
3. **Set the Next Target Condition**: What's the next achievable milestone toward the direction? Define it as a measurable outcome with a timeframe
4. **Experiment**: What's the next step to move from current condition toward the target? Design the smallest experiment. Run it. Measure results
5. **Repeat**: Update your understanding. Set a new target condition. Keep iterating

## Template
| Step | Description |
|------|-----------|
| Direction | Strategic goal / vision: |
| Current Condition | Metrics now: ___ Customer feedback: ___ Last experiment result: ___ |
| Target Condition | Next milestone: ___ Metric target: ___ By when: ___ |
| Next Experiment | What we'll try: ___ What we'll measure: ___ |

## Example
**Direction**: Become the default analytics tool for mid-market SaaS companies
**Current Condition**: 120 mid-market customers, 40% churn in first 90 days, primary complaint is "too complex to set up"
**Target Condition**: Reduce 90-day churn from 40% to 25% by end of Q2
**Experiment 1**: Pre-built dashboard templates for SaaS metrics → measure if setup time decreases and 30-day engagement improves
**Result**: Setup time dropped 50%, but engagement only improved 8% → need to also address data connection complexity
**Experiment 2**: One-click data source connectors for top 5 SaaS tools → measure connection completion rate

## Key Pitfalls
- Skipping "Grasp the Current Condition" — you can't improve what you don't measure
- Setting target conditions that are too ambitious — the next target should be achievable, not the end goal
- Not actually experimenting — teams revert to building large features instead of small tests
- Confusing the direction with the target condition — the direction is the vision; the target is the next step

## Related Frameworks
- **OKRs** — the Direction and Target Condition often map to Objectives and Key Results
- **Lean Startup** — Build-Measure-Learn is the experiment loop within Product Kata
- **Continuous Discovery** — provides the customer input for grasping the current condition
- **Feature Factory Antidote** — Product Kata is a remedy for feature factory syndrome
