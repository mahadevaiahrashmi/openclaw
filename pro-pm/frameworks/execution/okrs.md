# OKRs (Objectives and Key Results)

## What It Is
OKRs, popularized by John Doerr (Measure What Matters, 2018) and originated by Andy Grove at Intel, is a goal-setting framework that aligns teams around measurable outcomes. An Objective is a qualitative, ambitious goal. Key Results (2–5 per Objective) are specific, measurable outcomes that indicate the Objective is achieved. OKRs create alignment, focus, and transparency.

## When to Use
- Quarterly and annual goal-setting across the organization
- Aligning teams around outcomes instead of outputs
- When teams need clear direction while maintaining autonomy on *how* to achieve goals
- Tracking progress and creating accountability without micromanagement

## When NOT to Use
- Day-to-day task management — OKRs are strategic, not tactical
- As a performance review tool — OKRs should encourage ambition, not punish misses
- When the organization can't commit to the cadence and discipline required

## How to Apply
1. **Set Objectives**: Qualitative, inspiring, time-bound. Answer: "Where do we want to go?"
   - Good: "Become the most trusted analytics platform for mid-market"
   - Bad: "Increase revenue" (too vague) or "Ship feature X" (output, not outcome)
2. **Define Key Results** (2–5 per Objective): Quantitative, measurable. Answer: "How will we know we're getting there?"
   - Good: "Increase NPS from 32 to 50" or "Reduce time-to-value from 14 days to 3 days"
   - Bad: "Launch redesign" (binary output) or "Do customer research" (activity)
3. **Cascade and align**: Company OKRs → Team OKRs → Individual OKRs (optional). Ensure vertical and horizontal alignment
4. **Score at end of quarter**: 0.0–1.0 scale. Target 0.6–0.7 (stretch goals). 1.0 means it wasn't ambitious enough
5. **Review and learn**: What drove progress? What blocked it? Feed learnings into next quarter

## Template
**Objective**: [Qualitative, inspiring, time-bound]

| Key Result | Start | Target | Current | Score |
|-----------|-------|--------|---------|-------|
| KR1: | | | | /1.0 |
| KR2: | | | | /1.0 |
| KR3: | | | | /1.0 |

**Overall Objective Score**: ___

## Example
**Objective**: "Make our onboarding experience delightful for new users" (Q3)
- KR1: Increase 7-day activation rate from 25% to 45%
- KR2: Reduce median time-to-first-value from 8 minutes to 2 minutes
- KR3: Achieve onboarding NPS of 60+ (currently 38)
- KR4: Reduce onboarding support tickets by 40%

End of quarter: KR1 = 38% (0.6), KR2 = 4 min (0.5), KR3 = 55 (0.7), KR4 = -35% (0.8) → Average: 0.65 — healthy stretch

## Key Pitfalls
- Writing Key Results as tasks/outputs instead of outcomes — "Launch feature X" is not a KR
- Too many OKRs — 2–3 Objectives with 3–5 KRs each. More = no focus
- Not reviewing mid-quarter — OKRs need check-ins, not just end-of-quarter scoring
- Using OKRs for performance reviews — this discourages ambitious goal-setting
- Sandbagging — setting easy goals to ensure 1.0 scores

## Related Frameworks
- **North Star Framework** — defines the metric that OKRs should ladder up to
- **Impact Mapping** — connects OKRs to specific user behaviors and deliverables
- **RICE** — prioritizes the work that moves OKR needle most
- **Three Horizons** — OKRs should span horizons, not just H1
