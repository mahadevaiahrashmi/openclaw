# Activation Framework

## What It Is
Activation is the critical moment when a new user first experiences your product's core value — the "aha moment." The Activation Framework structures how to define, measure, and optimize this moment. It distinguishes between the Setup Moment (mechanical steps to get started) and the Aha Moment (when the user first perceives value). Activation is typically the highest-leverage growth metric to optimize.

## When to Use
- When signup-to-active-user conversion is low
- Defining what "activation" means for your product
- Optimizing the new user experience to deliver value faster
- When retention is strong for activated users but activation rate is poor

## When NOT to Use
- Products with immediate, obvious value that don't need onboarding optimization
- When retention is the primary problem (users activate but don't come back)
- Before you understand your core value proposition — define the "aha" first

## How to Apply
1. **Define the Aha Moment**: What action or experience makes users "get it"? Analyze your best retained users — what did they all do early on?
2. **Define the Setup Moment**: What mechanical steps must happen before the aha moment? (Account creation, data connection, profile setup)
3. **Map the path**: Setup → [friction points] → Aha Moment
4. **Measure activation rate**: % of signups who reach the aha moment within a defined timeframe
5. **Optimize**: Remove unnecessary setup steps. Guide users toward the aha moment faster. Use progressive onboarding (teach by doing, not tutorials)
6. **Validate**: Activated users should have significantly better retention than non-activated users. If not, your aha moment definition is wrong

## Example
**Product**: Analytics dashboard tool
- **Setup Moment**: Sign up → connect data source → wait for data sync
- **Aha Moment**: User sees their first automatically generated insight (not just raw data)
- **Current path**: 6 steps, takes ~25 minutes, 22% completion rate
- **Optimization**: Pre-populate with sample data so users see an insight in <2 minutes, then prompt to connect real data
- **Result**: Activation rate improved from 22% to 41%. D30 retention for activated users: 65% vs. 12% for non-activated

## Key Pitfalls
- Defining aha moment based on opinion instead of data — analyze what retained users actually did
- Making the setup too long before the aha — users should experience value as early as possible
- Conflating activation with signup — signing up is not activation
- Not measuring the correlation between activation and retention — if it's weak, your aha moment is wrong

## Related Frameworks
- **Pirate Metrics (AARRR)** — activation is the second stage
- **Hook Model** — the first hook cycle is essentially the activation moment
- **Product-Led Growth** — activation is the core of PLG's value-before-purchase model
- **Time-to-Value** — the metric that quantifies how fast activation happens
