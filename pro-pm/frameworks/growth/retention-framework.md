# Retention Framework

## What It Is
The Retention Framework segments user retention into three distinct categories — New User Retention, Current User Retention, and Resurrected User Retention — each requiring different strategies. Retention is measured through retention curves that show what percentage of a cohort returns over time. A flattening curve indicates product-market fit; a curve that trends to zero indicates a leaking bucket.

## When to Use
- Diagnosing why users don't come back
- Building a retention strategy with differentiated approaches for different user types
- Measuring product-market fit through retention curve shape
- When growth is stalling despite good acquisition — retention is likely the bottleneck

## When NOT to Use
- Products designed for one-time use (tax filing, home buying)
- Before you have enough cohort data to analyze (need at least 3–4 weeks of data)
- When activation is the primary problem — fix activation first, then retention

## How to Apply
1. **New User Retention** (Day 1–14): Did new users come back after their first session?
   - Focus: Onboarding, activation, first-session value delivery
   - Metric: D1, D3, D7 retention
   - Lever: Faster time-to-value, better onboarding, habit triggers
2. **Current User Retention** (Day 14+): Are activated users staying engaged?
   - Focus: Core product value, engagement loops, habit formation
   - Metric: D30, D60, D90 retention, weekly/monthly usage frequency
   - Lever: Feature depth, personalization, notifications, community
3. **Resurrected User Retention**: Are churned users coming back?
   - Focus: Win-back campaigns, product improvements since they left
   - Metric: Resurrection rate, re-engagement rate
   - Lever: Email campaigns, new feature announcements, incentives
4. **Analyze retention curves** by cohort: Plot % retained vs. time. Look for the curve to flatten (good) or approach zero (bad)
5. **Benchmark** against your product type and usage frequency

## Example
**Product**: Fitness tracking app
- **New User D1**: 45% → Problem: Users download but don't log their first workout
  - Fix: Add quick "log your last workout" during onboarding instead of requiring a new workout
- **Current D30**: 28% → Problem: Motivation drops after the novelty fades
  - Fix: Add social challenges, streak tracking, and weekly progress summaries
- **Resurrected**: 8% → Problem: Churned users forgot the app exists
  - Fix: Monthly email with "your friends are on a streak" + new feature highlights

## Key Pitfalls
- Treating all retention the same — new user retention requires different strategies than current user retention
- Looking only at aggregate retention instead of cohort-based analysis — aggregates hide trends
- Using retention as a vanity metric ("30% D30 retention!") without context — benchmark against your category
- Over-investing in resurrection when new user retention is the real problem
