# Growth Loops

## What It Is
Growth Loops, popularized by Reforge, replace the traditional linear funnel model with self-reinforcing cycles where the output of one cohort of users becomes the input for acquiring the next. Unlike funnels (which leak), loops compound. There are four main types: viral loops, content loops, paid loops, and sales loops. The strongest growth engines use multiple interconnected loops.

## When to Use
- Designing your core growth engine — how does growth compound?
- When linear acquisition channels are hitting diminishing returns
- Comparing growth strategies by their compounding potential
- Aligning product, marketing, and sales around a shared growth model

## When NOT to Use
- Very early-stage products without enough users to create loops
- Products where growth is purely sales-driven and self-serve isn't viable
- As a replacement for fixing activation/retention — loops amplify what works, they don't fix what's broken

## How to Apply
1. **Identify your existing loops** (or design new ones):
   - **Viral Loop**: Users invite others → new users join → they invite more (Slack, Dropbox)
   - **Content Loop**: Users create content → content gets indexed/shared → attracts new users (Pinterest, Stack Overflow)
   - **Paid Loop**: Revenue funds ads → ads acquire users → users generate revenue (Uber)
   - **Sales Loop**: Revenue funds sales team → sales closes deals → deals generate revenue → funds more sales
2. **Map the loop**: Draw each step. Identify the input, the action, and the output that becomes the next input
3. **Measure loop speed**: How long does one rotation take? (Cycle time)
4. **Measure loop efficiency**: What % of output becomes input for the next rotation? (Viral coefficient, content indexing rate, ROAS)
5. **Optimize**: Reduce friction at each step. Increase the output that feeds back in
6. **Layer loops**: Build multiple loops that reinforce each other

## Template
```
[New User Input] → [Step 1: Onboard] → [Step 2: Core Action] → [Step 3: Output/Share] → [Step 4: Discovery] → [New User Input]
```

## Example
**Product**: Design tool (Figma-like)
- **Viral Loop**: Designer creates a file → shares with collaborator → collaborator signs up → creates their own files → shares with others
- **Content Loop**: Designers publish templates → templates indexed by Google → new designers find templates → sign up to use them
- **Cycle Time**: Viral: ~5 days. Content: ~30 days
- **Efficiency**: Viral coefficient: 0.6 (each user brings 0.6 new users). Content: 15% of templates rank in top 10 results
- **Optimization**: Reduce sharing friction (public links), improve template SEO

## Key Pitfalls
- Confusing virality with a viral loop — viral content is one-time; a loop is self-sustaining
- Building loops before the core product works — loops amplify value, they don't create it
- Ignoring loop decay — loops slow down as markets saturate; plan for this
- Only tracking top-of-funnel — measure the full loop, including what feeds back
