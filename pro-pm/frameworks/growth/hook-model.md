# Hook Model

## What It Is
The Hook Model, created by Nir Eyal (Hooked, 2014), is a four-step framework for building habit-forming products: Trigger, Action, Variable Reward, and Investment. The cycle repeats, deepening engagement over time. It's based on behavioral psychology research on habit formation and is designed to create products users return to without external prompting.

## When to Use
- Designing products where habitual engagement is critical (social, productivity, health)
- Improving retention by strengthening the habit loop
- Analyzing why competitor products are "sticky" and yours isn't
- Evaluating new feature ideas through a behavioral lens

## When NOT to Use
- Products used infrequently by design (insurance, real estate) — habits aren't the right lens
- When engagement is already strong and you need to focus on monetization or acquisition
- Without ethical consideration — the framework can be used manipulatively

## How to Apply
1. **Trigger**: What prompts the user to take action?
   - **External**: Notifications, emails, ads, word of mouth
   - **Internal**: Emotions (boredom, loneliness, uncertainty, FOMO) — the strongest triggers
2. **Action**: The simplest behavior done in anticipation of a reward. Must be easy (Fogg's model: Motivation + Ability + Trigger)
3. **Variable Reward**: The payoff, but unpredictable. Three types:
   - **Tribe**: Social rewards (likes, comments, validation)
   - **Hunt**: Material rewards (information, deals, content)
   - **Self**: Personal mastery, completion, competence
4. **Investment**: User puts something in (data, content, reputation, effort) that makes the next cycle more valuable
5. **Repeat**: The investment creates the trigger for the next loop

## Example
**Product**: Slack
- **Trigger**: External = notification badge. Internal = "Did someone message me?" (uncertainty/FOMO)
- **Action**: Open Slack, read message (extremely low effort — tap notification)
- **Variable Reward**: Tribe (team banter, recognition) + Hunt (important information, decisions)
- **Investment**: Channels joined, message history, integrations set up, status customized → increases switching cost and trigger likelihood

## Key Pitfalls
- Designing for manipulation instead of genuine value — Eyal's "manipulation matrix" asks: does this improve the user's life?
- Forcing variable rewards that feel artificial — rewards must be genuine, not gamification for its own sake
- Neglecting the Investment step — without investment, there's no increasing returns and users leave easily
- Applying to products where habit formation isn't appropriate or ethical
