# SCQA (Situation-Complication-Question-Answer)

## What It Is
SCQA is a narrative structure from McKinsey (rooted in Barbara Minto's work) that builds a compelling case by setting context before delivering the recommendation. It's the storytelling complement to the Pyramid Principle: Situation (what's happening), Complication (why it's a problem), Question (what should we do), Answer (the recommendation). It creates urgency and buy-in.

## When to Use
- Persuasive communications where you need to build a case (not just state a conclusion)
- Product strategy presentations to leadership
- Change management communications where context is needed for buy-in
- When stakeholders need to understand *why* before they'll accept *what*

## When NOT to Use
- Status updates where the audience already has context — use Pyramid Principle directly
- Urgent communications — skip the setup and state the action needed
- When the situation and complication are already well-known by the audience

## How to Apply
1. **Situation**: Set the context. What is the current state that everyone agrees on? (Non-controversial, establishes shared understanding)
2. **Complication**: What changed? What's the problem? Why can't we continue as-is? (Creates tension and urgency)
3. **Question**: What's the key question this raises? (Often implicit, but framing it explicitly sharpens focus)
4. **Answer**: Your recommendation. Followed by supporting arguments (Pyramid structure)

## Template
**Situation**: [Agreed-upon context — 1-2 sentences]

**Complication**: [What changed or what's wrong — 1-2 sentences]

**Question**: [The key question this raises]

**Answer**: [Your recommendation + supporting arguments]

## Example
**Situation**: Our enterprise pipeline has grown 3x in the last 6 months, and we're winning 40% of deals in our core mid-market segment.

**Complication**: However, our win rate drops to 8% for companies with >5,000 employees because they require FedRAMP compliance, which we don't have. We're losing $4.2M in qualified pipeline per quarter to this single gap.

**Question**: Should we invest in FedRAMP certification, or continue focusing on mid-market where we're already winning?

**Answer**: We should begin FedRAMP certification this quarter. Three reasons: (1) $4.2M/quarter pipeline justifies the $600K investment in 12 months. (2) FedRAMP creates a competitive moat — only 3 competitors have it. (3) Government/large enterprise customers have 2x higher retention and 3x expansion rates, making LTV significantly better.

## Key Pitfalls
- Situation that's too long — keep it to 1–2 sentences; the audience likely knows the context
- Complication that isn't compelling — if there's no tension, the recommendation feels unjustified
- Skipping the Question — explicitly stating it focuses the audience on what you're about to answer
- Answer without supporting evidence — SCQA sets up the recommendation; Pyramid delivers it

## Related Frameworks
- **Pyramid Principle** — the Answer section should follow pyramid structure
- **Narrative Memo** — longer-form narrative that can use SCQA as its backbone
- **Working Backwards** — Amazon's press release is a different narrative structure for the same purpose
- **Stakeholder Influence** — SCQA is a tool for influencing stakeholder decisions
