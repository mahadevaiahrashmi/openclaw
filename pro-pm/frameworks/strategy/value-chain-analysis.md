# Value Chain Analysis

## What It Is
Value Chain Analysis, created by Michael Porter (1985), decomposes a company's activities into primary and support activities to identify where value is created and where competitive advantage can be built. Primary activities (inbound logistics, operations, outbound logistics, marketing/sales, service) directly create value. Support activities (infrastructure, HR, technology, procurement) enable the primary ones.

## When to Use
- Identifying which activities create the most value and where to invest
- Finding cost reduction opportunities without sacrificing differentiation
- Analyzing competitors' value chains to find strategic gaps
- Make vs. buy decisions for specific value chain components

## When NOT to Use
- Software products where Porter's manufacturing-era categories don't map cleanly — adapt the framework
- Feature-level prioritization — too high-level for sprint planning
- When you need customer-facing analysis — value chain is internally focused

## How to Apply
1. **Map primary activities** (adapted for software/SaaS):
   - **Inbound**: Data ingestion, API integrations, content acquisition
   - **Operations**: Core product development, infrastructure, deployment
   - **Outbound**: Distribution, delivery, platform availability
   - **Marketing & Sales**: Acquisition, positioning, sales process
   - **Service**: Support, success, onboarding, retention
2. **Map support activities**: Infrastructure (legal, finance), HR, Technology (internal tools), Procurement
3. **Assess each activity**: Cost, value created, differentiation potential
4. **Identify linkages**: Where does excellence in one activity amplify another?
5. **Compare to competitors**: Where are you stronger/weaker in the chain?
6. **Decide**: Where to invest, outsource, or eliminate

## Example
**Company**: SaaS analytics platform
- **Operations** (core product): Strong — proprietary query engine is a differentiator. Invest more
- **Marketing**: Weak — content marketing underperforms competitors. Hire or outsource
- **Service**: Average — onboarding is good, but support response times lag. Improve
- **Technology (support)**: Over-invested in internal tooling — consolidate to off-the-shelf

## Key Pitfalls
- Applying Porter's manufacturing categories literally to software — adapt the categories
- Treating activities as independent — the linkages between activities create the most value
- Focusing only on cost reduction — value chain analysis is also about identifying differentiation
- Not comparing to competitors — the analysis is most useful as a relative comparison

## Related Frameworks
- **Wardley Mapping** — maps components by evolution, complementing value chain analysis
- **Porter's Five Forces** — industry-level view that pairs with internal value chain
- **Business Model Canvas** — broader view including revenue, customers, and partnerships
- **BCG Matrix** — portfolio-level decisions informed by value chain strengths
