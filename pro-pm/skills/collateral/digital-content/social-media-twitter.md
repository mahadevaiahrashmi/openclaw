# Social Media — Twitter/X

## Purpose
Creates Twitter/X content in multiple formats—thread format, single tweet variants, launch tweets, and engagement tweets. Concise, punchy, and value-packed. Designed for maximum reach and engagement within character constraints.

## Auto-Trigger Patterns
- "Write a Twitter thread about [topic]"
- "Create tweets for [launch/announcement]"
- "I need Twitter content for [campaign]"
- "Help me write a tweet thread on [subject]"

## Inputs

**Zero-setup:** Only the user prompt is required. If context files are empty, use `context/_defaults.md` and label assumptions. See `skills/_GLOBAL-BEHAVIOR.md`.

- **Required:** Topic, key message, 3-5 supporting points for threads
- **Optional:** Links to include, images/video descriptions, launch timing, engagement strategy
- **Reference files:** product-brief.md, blog-post-announcement.md, social-media-linkedin.md

## Process
1. **Choose the format** — Single tweet (announcements), thread (education/storytelling), or engagement tweet (questions/polls).
2. **Nail the hook** — First tweet determines if anyone reads the rest. Make it irresistible.
3. **Optimize for value density** — Every word matters. Cut ruthlessly.
4. **Thread structure** — Hook → context → key points → takeaway → CTA. Each tweet standalone-readable.
5. **Add engagement mechanics** — Retweet prompts, questions, save-worthy content.

## Output Format
```
# Twitter Content: [Topic]

## Thread (5-10 tweets)
🧵 1/ [Hook — bold claim, surprising stat, or question]

2/ [Context setting — why this matters]

3/ [Key point 1 — one idea, clearly stated]

4/ [Key point 2 — with specific example or data]

5/ [Key point 3 — the insight most people miss]

6/ [Takeaway — the actionable conclusion]

7/ [CTA — follow, bookmark, link, reply]

---

## Single Tweet Variants

### Launch Tweet
[Product announcement. What + impact. Link.]

### Engagement Tweet
[Question or hot take designed to drive replies]

### Metrics Tweet
[Specific, surprising number with context]

### Quote Tweet Template
[Framework for QTing relevant industry content]
```

## Quality Standards
- Hook tweet works as a standalone piece of content.
- Each thread tweet can be retweeted independently and still make sense.
- Value per character is maximized—no filler words.
- **Anti-patterns:** Thread that's a blog post chopped up, generic announcements, too many hashtags, self-congratulatory tone.

## Audience
Tech-savvy professionals, developers, industry practitioners, and potential customers who use Twitter for professional learning and networking. They follow high-signal accounts and unfollow noise.

## Formatting Guidelines
- 280 characters max per tweet. Aim for 200-240 for readability.
- Use line breaks within tweets for readability.
- 1-2 emojis max per tweet (or none—depends on brand).
- Thread numbering: "1/" format. Add 🧵 to first tweet.
- Hashtags: 0-2 per tweet. Never more.

## Example
**Thread hook:** "🧵 1/ We analyzed 500 failed product launches to find the #1 reason products die.\n\nIt's not competition. It's not funding. It's not bad engineering.\n\nIt's something much simpler (and fixable):"

**Launch tweet:** "Shipping today: AI-powered search across all your connected tools.\n\n→ 1.2s average query time\n→ Search across 50+ integrations\n→ No configuration required\n\nYour team just got 3 hours back per week.\n\nTry it free ↓"
